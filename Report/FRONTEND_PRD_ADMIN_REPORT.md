## 📐 와이어프레임 분석 결과

### 전체 레이아웃 구조

와이어프레임 분석 결과, 관리자 메인 화면은 다음과 같이 구성됩니다:

1. **헤더 영역**
   - 왼쪽: "COZY" 로고 (어두운 회색 박스, 하단 초록색 테두리)
   - 오른쪽: "주문하기" 버튼, "관리자" 텍스트/버튼 (활성 상태 표시 - 어두운 테두리)

2. **관리자 대시보드 섹션**
   - 주문 통계 표시
   - 형식: "총 주문 1 / 주문 접수 1 / 제조 중 0 / 제조 완료 0"
   - 한 줄에 모든 통계 표시

3. **재고 현황 섹션**
   - 각 메뉴별 재고 카드 (3열 그리드)
   - 메뉴명, 재고 수량, 증감 버튼 (+ / -)
   - 예시: 아메리카노(ICE), 아메리카노(HOT), 카페라떼

4. **주문 현황 섹션**
   - 주문 목록 카드 형태로 표시
   - 각 주문 카드: 일시, 메뉴, 금액, "주문 접수" 버튼

### 디자인 특징

- **색상**: 흰색 배경, 회색 텍스트, 초록색 강조 (로고 하단 테두리)
- **레이아웃**: 섹션별 둥근 모서리 박스로 구분
- **버튼**: 연한 회색 배경, 회색 테두리
- **활성 상태**: 관리자 버튼에 어두운 테두리로 현재 화면 표시

## 📄 PRD 문서 구성

### 1. 개요 섹션

- 문서 정보 (버전, 작성일, 대상)
- 기술 스택 (React, React Router, Context API/Zustand, Axios, Tailwind CSS)
- 사용자 역할 (관리자: 매장 운영자, 프랜차이즈 관리자)

### 2. 와이어프레임 분석 섹션

- 전체 레이아웃 구조 (ASCII 다이어그램)
- 주요 디자인 특징 (색상 스키마, 레이아웃, 타이포그래피)

### 3. 화면 구조 섹션

전체 네비게이션 구조:
- 관리자 대시보드 (`/admin`)
- 주문 관리 (`/admin/orders`)
- 메뉴 관리 (`/admin/menus`)
- 통계 (`/admin/statistics`)

### 4. 페이지별 상세 요구사항

#### 4.1 관리자 대시보드 (메인 화면)

**섹션 1: 관리자 대시보드 (주문 통계)**
- 통계 표시 형식: "총 주문 N / 주문 접수 N / 제조 중 N / 제조 완료 N"
- 실시간 업데이트 요구사항
- 디자인 스펙 (둥근 모서리, 패딩 등)

**섹션 2: 재고 현황**
- 재고 카드 구조 (3열 그리드)
- 재고 조정 기능 (+ / - 버튼)
- 즉시 서버 반영 및 피드백
- 재고 부족 경고 (5개 이하 시 빨간색)

**섹션 3: 주문 현황**
- 주문 목록 카드 형태
- 주문 정보: 일시, 항목, 금액
- 액션 버튼: "주문 접수" → "제조 중" → "제조 완료"
- 실시간 업데이트

#### 4.2 주문 관리 페이지

- 주문 목록 페이지 (`/admin/orders`)
- 주문 상세 페이지 (`/admin/orders/:id`)
- 정렬 및 필터 기능
- 상태 변경 기능

#### 4.3 메뉴 관리 페이지

- 메뉴 목록 (`/admin/menus`)
- 메뉴 등록 (`/admin/menus/new`)
- 메뉴 수정 (`/admin/menus/:id/edit`)
- 메뉴 삭제 및 판매 가능 여부 토글

#### 4.4 통계 페이지

- 매출 통계 (일별/주별/월별)
- 주문 통계 (건수, 평균 금액, 시간대별 분포)
- 인기 메뉴 통계

### 5. 공통 컴포넌트

#### 5.1 Header (헤더) - 관리자 화면

- 와이어프레임 기반 레이아웃
- 로고 영역 (왼쪽): "COZY" 텍스트, 초록색 하단 테두리
- 액션 버튼 (오른쪽): "주문하기", "관리자" (활성 상태 표시)

#### 5.2 통계 카드 컴포넌트

- 주문 통계 표시
- 둥근 모서리, 회색 테두리

#### 5.3 재고 카드 컴포넌트

- 메뉴명, 재고 수량, 증감 버튼
- 독립적인 카드 형태

#### 5.4 주문 카드 컴포넌트

- 주문 정보 표시
- 액션 버튼 오른쪽 정렬

### 6. 관리자 플로우

- **플로우 1**: 주문 처리 (접수 → 제조 중 → 제조 완료)
- **플로우 2**: 재고 관리 (재고 확인 → 증감 조정)
- **플로우 3**: 메뉴 관리 (목록 → 등록/수정)

### 7. UI/UX 가이드라인

- 디자인 원칙 (명확성, 효율성, 피드백)
- 색상 팔레트 (와이어프레임 기반)
- 상태 표시 (접수 대기, 제조 중, 제조 완료, 재고 부족)

### 8. 반응형 디자인

### 차이점

| 항목 | 사용자 화면 | 관리자 화면 |

| **주요 기능** | 메뉴 조회, 주문하기, 주문 내역 | 주문 관리, 재고 관리, 통계 |
