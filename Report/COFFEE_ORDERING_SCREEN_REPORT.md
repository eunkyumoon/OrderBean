# 커피 주문하기 화면 구현 보고서

## 작업 개요
와이어프레임 이미지를 참고하여 커피 주문하기 화면을 구현했습니다.

## 작업 일시
2024년 (현재 날짜)

## 구현 내용

### 1. 화면 구조
와이어프레임에 따라 다음 3개의 주요 섹션으로 구성되었습니다:

#### 1.1 헤더 영역
- **위치**: 화면 상단
- **구성 요소**:
  - 왼쪽: "OrderBean - 커피 주문" 로고
  - 오른쪽: "주문하기", "관리자" 네비게이션 버튼
- **구현 파일**: `frontend/src/components/Header.jsx`

#### 1.2 메뉴 표시 영역
- **위치**: 화면 중앙
- **구성 요소**:
  - 3열 그리드 레이아웃으로 메뉴 카드 표시
  - 각 메뉴 카드 포함 요소:
    - 이미지 플레이스홀더 (X 표시)
    - 메뉴명
    - 기본 가격
    - 간단한 설명
    - 옵션 체크박스 (샷 추가 +500원, 시럽 추가 +0원)
    - "담기" 버튼
- **구현 파일**: 
  - `frontend/src/pages/MenuList.jsx`
  - `frontend/src/components/MenuCard.jsx`

#### 1.3 장바구니 영역
- **위치**: 화면 하단 고정
- **구성 요소**:
  - 왼쪽: 주문 내역 리스트
    - 메뉴명 (옵션) X 수량 형식
    - 각 항목의 가격 (오른쪽 정렬)
  - 오른쪽: 총 금액 및 "주문하기" 버튼
- **구현 파일**: `frontend/src/components/Cart.jsx`

### 2. 주요 기능 구현

#### 2.1 메뉴 표시
- API에서 메뉴 데이터를 가져와 표시
- API 실패 시 기본 메뉴 데이터 사용
- 와이어프레임에 표시된 메뉴 포함:
  - 아메리카노(ICE) - 4,000원
  - 아메리카노(HOT) - 4,000원
  - 카페라떼 - 5,000원

#### 2.2 옵션 선택
- 각 메뉴 카드에서 옵션 선택 가능:
  - 샷 추가 (+500원)
  - 시럽 추가 (+0원)
- 체크박스로 옵션 선택/해제

#### 2.3 장바구니 관리
- "담기" 버튼 클릭 시 장바구니에 추가
- 동일한 메뉴와 옵션 조합이 이미 장바구니에 있으면 수량 증가
- 장바구니 항목 제거 기능
- 실시간 총 금액 계산

#### 2.4 주문 처리
- "주문하기" 버튼 클릭 시 주문 완료 알림
- 주문 후 장바구니 초기화

### 3. 스타일링

#### 3.1 레이아웃
- 반응형 디자인 적용
- 데스크톱: 3열 그리드
- 태블릿: 2열 그리드
- 모바일: 1열 그리드

#### 3.2 색상 및 디자인
- 깔끔한 흰색 배경
- 회색 테두리와 그림자 효과
- 버튼 호버 효과
- 장바구니 하단 고정 및 그림자 효과

#### 3.3 이미지 플레이스홀더
- 와이어프레임에 맞게 X 표시 구현
- CSS로 대각선 X 표시 생성

### 4. 수정된 파일 목록

1. **`frontend/src/index.jsx`**
   - 실제 App 컴포넌트 렌더링으로 변경
   - 테스트 코드 제거

2. **`frontend/src/styles/App.css`**
   - 이미지 플레이스홀더 X 표시 스타일 개선
   - 장바구니 레이아웃 조정
   - 메뉴 리스트 하단 여백 추가 (장바구니 공간 확보)

3. **`frontend/src/components/MenuCard.jsx`**
   - 이미지 플레이스홀더 표시 방식 개선

4. **`frontend/src/components/Cart.jsx`**
   - 장바구니 항목 표시 형식 변경
   - "메뉴명 (옵션) X 수량" 형식으로 통일
   - 가격 오른쪽 정렬

### 5. 기술 스택
- **React**: UI 컴포넌트 구성
- **React Router**: 페이지 라우팅
- **CSS3**: 스타일링 및 레이아웃
- **Vite**: 개발 서버 및 빌드 도구
