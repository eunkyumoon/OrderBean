## 와이어프레임 분석

### 전체 레이아웃 구조

와이어프레임에 따르면 사용자 메인 화면은 다음과 같이 구성됩니다:

```
┌─────────────────────────────────────────┐
│ Header                                  │
│ [COZY 로고]        [주문하기] [관리자]  │
├─────────────────────────────────────────┤
│                                         │
│ 메뉴 카드 영역 (3개 가로 배치)          │
│ ┌──────┐ ┌──────┐ ┌──────┐           │
│ │카드1 │ │카드2 │ │카드3 │           │
│ └──────┘ └──────┘ └──────┘           │
│                                         │
├─────────────────────────────────────────┤
│ 장바구니 영역                            │
│ ┌─────────────────────────────────────┐ │
│ │ 장바구니                             │ │
│ │ 항목1: 아메리카노(ICE) (샷 추가)    │ │
│ │ 항목2: 아메리카노(HOT) X 2          │ │
│ │ 총 금액: 12,500원                   │ │
│ │ [주문하기]                          │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

### 주요 디자인 특징

1. **색상 스키마**
   - 주 색상: 흰색 배경, 회색 텍스트
   - 강조 색상: 초록색 (로고 하단 테두리)
   - 버튼: 연한 회색 배경, 회색 테두리

2. **레이아웃**
   - 상단: 헤더 (로고 + 액션 버튼)
   - 중간: 메뉴 카드 그리드 (3열)
   - 하단: 장바구니 (고정 영역)

3. **타이포그래피**
   - 로고: 굵은 글씨, 어두운 회색
   - 메뉴명: 굵은 글씨
   - 가격: 일반 글씨
   - 총 금액: 굵은 글씨로 강조

## 화면 구조

### 전체 네비게이션 구조

```
사용자 화면
├── 랜딩 페이지 (/)
├── 로그인/회원가입 (/auth)
│   ├── 로그인 (/auth/login)
│   └── 회원가입 (/auth/register)
├── 메인 화면 (/home)
│   ├── 메뉴 목록 (/menus)
│   ├── 메뉴 상세 (/menus/:id)
│   └── 장바구니 (/cart)
├── 주문 화면 (/orders)
│   ├── 주문 생성 (/orders/new)
│   ├── 주문 내역 (/orders)
│   └── 주문 상세 (/orders/:id)
├── 레시피 화면 (/recipes)
│   ├── 레시피 목록 (/recipes)
│   ├── 레시피 생성 (/recipes/new)
│   └── 레시피 상세 (/recipes/:id)
└── 프로필 (/profile)
```

## 페이지별 상세 요구사항

### 1. 랜딩 페이지 (Landing Page)

#### 목적
- 서비스 소개 및 첫 방문자 유도

#### 주요 요소

**헤더 (Header)**
- 로고: "OrderBean" 브랜드명
- 네비게이션 메뉴
  - 메뉴 보기
  - 로그인
  - 회원가입

**히어로 섹션 (Hero Section)**
- 서비스 소개 문구
- 주요 기능 요약
- "시작하기" CTA 버튼

**기능 소개 섹션**
- 빠른 주문
- 커스터마이징
- 재주문 기능

**푸터 (Footer)**
- 연락처 정보
- 이용약관 링크
- 개인정보처리방침 링크

#### 요구사항
- 모바일 반응형 디자인
- 빠른 로딩 속도
- 명확한 CTA (Call to Action)

### 2. 로그인/회원가입 페이지

#### 경로
- `/auth/login` - 로그인
- `/auth/register` - 회원가입

#### 목적
- 사용자 인증 및 계정 생성

#### 로그인 페이지 (`/auth/login`)

**요구사항**
- 이메일 입력 필드
- 비밀번호 입력 필드
- "로그인" 버튼
- "회원가입" 링크
- "비밀번호 찾기" 링크 (선택사항)

**검증 규칙**
- 이메일 형식 검증
- 비밀번호 필수 입력
- 에러 메시지 표시

**상태 관리**
- 로그인 성공 시 JWT 토큰 저장
- 로그인 상태에 따른 리다이렉트

#### 회원가입 페이지 (`/auth/register`)

**요구사항**
- 이메일 입력 필드
- 비밀번호 입력 필드
- 비밀번호 확인 필드
- 이름 입력 필드
- 전화번호 입력 필드 (선택)
- "회원가입" 버튼
- "로그인" 링크

**검증 규칙**
- 이메일 중복 확인
- 비밀번호 일치 확인
- 비밀번호 강도 검증 (최소 8자)
- 실시간 유효성 검사

- 장바구니 관리

**헤더 컴포넌트**

- **왼쪽**: "COZY" 로고 (어두운 회색 박스, 하단 초록색 테두리)
- **오른쪽**: 
  - "주문하기" 버튼 (연한 회색 배경, 회색 테두리)
  - "관리자" 텍스트 링크

**메뉴 카드 컴포넌트** (와이어프레임 기반)

각 카드 구조:
1. **이미지 영역**
   - 상단에 이미지 플레이스홀더 (대각선 X 표시)
   - 실제 이미지 또는 플레이스홀더

2. **메뉴 정보**
   - 메뉴명 (굵은 글씨, 어두운 회색)
   - 가격 (일반 글씨)
   - 간단한 설명 ("간단한 설명..." 텍스트)

3. **옵션 체크박스**
   - "샷 추가 (+500원)" 체크박스
   - "시럽 추가 (+0원)" 체크박스
   - 체크박스는 각 카드 내에서 독립적으로 작동

4. **액션 버튼**
   - "담기" 버튼 (연한 회색 배경, 회색 테두리)
   - 카드 하단 중앙 배치

**장바구니 컴포넌트** (하단 고정)

1. **제목**
   - "장바구니" 텍스트 (왼쪽 상단)

2. **주문 항목 목록**
   - 각 항목 표시:
     - 왼쪽: 메뉴명 + 옵션 정보 + 수량
     - 오른쪽: 항목별 가격
   - 예시:
     - "아메리카노(ICE) (샷 추가) X 1" → "4,500원"
     - "아메리카노(HOT) X 2" → "8,000원"

3. **총 금액**
   - "총 금액: 12,500원" (굵은 글씨, 오른쪽 정렬)

4. **주문하기 버튼**
   - "주문하기" 버튼 (연한 회색 배경, 회색 테두리)
   - 장바구니 하단 중앙 또는 오른쪽 배치

**기능 요구사항** (와이어프레임 기반)

1. **메뉴 카드 그리드**
   - 3열 그리드 레이아웃 (데스크톱)
   - 각 카드는 독립적으로 작동
   - 스크롤 가능 (메뉴가 많을 경우)

2. **옵션 선택 (카드 내)**
   - 각 메뉴 카드에서 직접 옵션 선택
   - 체크박스로 옵션 활성화/비활성화
   - 옵션 선택 시 가격 자동 계산 (백그라운드)
   - "샷 추가": +500원
   - "시럽 추가": +0원 (또는 설정된 가격)

3. **장바구니에 담기**
   - "담기" 버튼 클릭 시
   - 선택한 옵션과 함께 장바구니에 추가
   - 장바구니 하단 영역에 즉시 반영
   - 중복 메뉴는 수량 증가 또는 별도 항목으로 추가

4. **장바구니 관리**
   - 하단 고정 영역에 항상 표시
   - 항목별 가격 표시
   - 총 금액 자동 계산
   - 실시간 업데이트

5. **주문하기**
   - "주문하기" 버튼 클릭 시
   - 주문 확인 페이지로 이동 (또는 모달)
   - 결제 프로세스 시작

**상태 관리**
- 메뉴 목록 데이터 (React Query 캐싱)
- 각 카드의 옵션 선택 상태 (로컬 상태)
- 장바구니 항목 (전역 상태)
- 총 금액 (계산된 값)

### 4. 메뉴 상세 페이지 (선택사항)

> **참고**: 와이어프레임에서는 메인 화면에서 직접 옵션 선택 및 담기가 가능하도록 설계되어 있습니다. 메뉴 상세 페이지는 선택적 기능으로, 향후 확장 시 구현할 수 있습니다.

#### 경로
- `/menus/:id` - 메뉴 상세 (선택사항)

#### 목적
- 메뉴 상세 정보 확인 및 추가 커스터마이징 옵션 선택

#### 와이어프레임 기반 설계 변경

와이어프레임 분석 결과, **메인 화면에서 직접 옵션 선택 및 담기가 가능**하므로:
- 메뉴 상세 페이지는 선택사항
- 메인 화면의 카드에서 직접 옵션 선택 후 담기 가능
- 상세 페이지는 추가 정보나 복잡한 옵션이 필요할 때만 사용

### 5. 장바구니 (하단 고정 영역)

#### 위치
- 메인 화면 하단 고정 영역

#### 목적
- 주문할 메뉴 확인 및 주문 진행

#### 레이아웃 구조 (와이어프레임 기반)

```
┌─────────────────────────────────────────┐
│ 장바구니                                 │
│                                         │
│ 아메리카노(ICE) (샷 추가) X 1           │
│                           4,500원      │
│                                         │
│ 아메리카노(HOT) X 2                     │
│                           8,000원      │
│                                         │
│ 총 금액: 12,500원                      │
│                                         │
│              [주문하기]                 │
└─────────────────────────────────────────┘
```

**디자인 요구사항**

1. **레이아웃**
   - 하단 고정 영역 (Sticky Footer)
   - 연한 회색 배경
   - 회색 테두리
   - 화면 하단에 항상 표시

2. **제목**
   - "장바구니" 텍스트
   - 왼쪽 상단 배치
   - 굵은 글씨 또는 일반 글씨

3. **주문 항목 표시**
   - 각 항목을 한 줄로 표시
   - 왼쪽: 메뉴명 + 옵션 정보 + 수량
   - 오른쪽: 항목별 가격
   - 항목 간 구분선 또는 간격

4. **총 금액**
   - "총 금액: [금액]원" 형식
   - 굵은 글씨로 강조
   - 오른쪽 정렬 또는 중앙 정렬

5. **주문하기 버튼**
   - "주문하기" 텍스트
   - 연한 회색 배경
   - 회색 테두리
   - 중앙 또는 오른쪽 배치

**기능 요구사항**

1. **실시간 업데이트**
   - 메뉴 카드에서 "담기" 클릭 시 즉시 반영
   - 항목 추가/제거 시 자동 업데이트
   - 총 금액 자동 재계산

2. **항목 표시 형식**
   - 메뉴명 + 옵션 정보 (괄호 안)
   - 수량 표시 (X N)
   - 항목별 가격 (오른쪽 정렬)

3. **빈 장바구니 상태**
   - 항목이 없을 때 "장바구니가 비어있습니다" 메시지

     - 회색 테두리
     - 클릭 시 주문 확인/결제 페이지로 이동
   - "관리자" 텍스트 링크
     - 일반 텍스트 스타일
     - 클릭 시 관리자 페이지로 이동 (권한 확인)

- 높이: 약 60-80px
- 배경: 흰색 또는 연한 회색
- 로고 하단 테두리: 초록색 (#28A745 또는 유사), 2-3px
- 버튼 스타일: 연한 회색 배경, 회색 테두리, 둥근 모서리 (선택)

**반응형**:
- 모바일: 로고 크기 축소, 버튼 텍스트 간소화 또는 아이콘

   - 색상만으로 정보 전달하지 않음

## 와이어프레임 구현 상세

### 메뉴 카드 컴포넌트 상세 스펙

#### 구조
```
┌─────────────────────┐
│ [이미지 플레이스홀더]│ (대각선 X 표시)
│                     │
│ 메뉴명 (굵은 글씨)  │
│ 가격                │
│ 간단한 설명...      │
│                     │
│ ☐ 샷 추가 (+500원)  │
│ ☐ 시럽 추가 (+0원)  │
│                     │
│    [담기]           │
└─────────────────────┘
```

#### 스타일 가이드

1. **카드 컨테이너**
   - 배경: 흰색
   - 테두리: 회색, 1px (선택사항)
   - 둥근 모서리: 8px (선택사항)
   - 그림자: 약한 그림자 (선택사항)
   - 패딩: 16px

2. **이미지 영역**
   - 높이: 150-200px
   - 배경: 연한 회색 (#F5F5F5)
   - 플레이스홀더: 대각선 X 표시 또는 이미지 아이콘
   - 비율: 16:9 또는 1:1

3. **텍스트 스타일**
   - 메뉴명: 18px, Bold, 어두운 회색
   - 가격: 16px, Regular, 어두운 회색
   - 설명: 14px, Regular, 중간 회색

4. **옵션 체크박스**
   - 체크박스 스타일: 기본 HTML 체크박스 또는 커스텀
   - 레이블: 14px, Regular
   - 가격 표시: 14px, Regular, 중간 회색
   - 간격: 옵션 간 8px

5. **담기 버튼**
   - 배경: 연한 회색 (#F5F5F5)
   - 테두리: 회색, 1px
   - 텍스트: 14px, Medium
   - 패딩: 8px 16px
   - 너비: 카드 너비의 80-90%

### 장바구니 컴포넌트 상세 스펙

#### 구조
```
┌─────────────────────────────────┐
│ 장바구니                         │
│                                 │
│ 항목1명 (옵션) X 수량    가격   │
│ 항목2명 X 수량           가격   │
│                                 │
│ 총 금액: 12,500원              │
│                                 │
│         [주문하기]              │
└─────────────────────────────────┘
```

#### 스타일 가이드

1. **컨테이너**
   - 배경: 연한 회색 (#F8F9FA)
   - 테두리: 회색, 1px
   - 패딩: 16px
   - 위치: 하단 고정 (position: sticky 또는 fixed)

2. **항목 표시**
   - 레이아웃: Flexbox (space-between)
   - 왼쪽: 메뉴명 + 옵션 + 수량
   - 오른쪽: 가격
   - 항목 간 간격: 12px
   - 폰트: 14px, Regular

3. **총 금액**
   - 폰트: 16px, Bold
   - 정렬: 오른쪽 또는 중앙
   - 상단 여백: 16px

4. **주문하기 버튼**
   - 배경: 연한 회색 (#F5F5F5)
   - 테두리: 회색, 1px
   - 텍스트: 16px, Medium
   - 패딩: 12px 24px
   - 너비: 100% 또는 적절한 너비
