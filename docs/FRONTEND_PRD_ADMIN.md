- **문서명**: OrderBean 프론트엔드 UI PRD - 관리자 화면

## 와이어프레임 분석

### 전체 레이아웃 구조

와이어프레임에 따르면 관리자 메인 화면은 다음과 같이 구성됩니다:

```
┌─────────────────────────────────────────┐
│ Header                                  │
│ [COZY 로고]        [주문하기] [관리자]  │ (관리자 활성)
├─────────────────────────────────────────┤
│ 관리자 대시보드                          │
│ ┌─────────────────────────────────────┐ │
│ │ 총 주문 1 / 주문 접수 1 /            │ │
│ │ 제조 중 0 / 제조 완료 0              │ │
│ └─────────────────────────────────────┘ │
├─────────────────────────────────────────┤
│ 재고 현황                               │
│ ┌──────┐ ┌──────┐ ┌──────┐           │
│ │아메리│ │아메리│ │카페  │           │
│ │카노  │ │카노  │ │라떼  │           │
│ │(ICE) │ │(HOT) │ │      │           │
│ │10개  │ │10개  │ │10개  │           │
│ │[+][-]│ │[+][-]│ │[+][-]│           │
│ └──────┘ └──────┘ └──────┘           │
├─────────────────────────────────────────┤
│ 주문 현황                               │
│ ┌─────────────────────────────────────┐ │
│ │ 7월 31일 13:00                       │ │
│ │ 아메리카노(ICE) x 1                  │ │
│ │ 4,000원              [주문 접수]    │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

### 주요 디자인 특징

1. **색상 스키마**
   - 주 색상: 흰색 배경, 회색 텍스트
   - 강조 색상: 초록색 (로고 하단 테두리)
   - 버튼: 연한 회색 배경, 회색 테두리
   - 활성 상태: 관리자 버튼에 어두운 테두리

2. **레이아웃**
   - 상단: 헤더 (로고 + 액션 버튼)
   - 중간: 섹션별 박스 레이아웃
   - 각 섹션: 둥근 모서리 박스로 구분

3. **타이포그래피**
   - 섹션 제목: 굵은 글씨 또는 일반 글씨
   - 통계 숫자: 굵은 글씨로 강조
   - 주문 정보: 일반 글씨

## 페이지별 상세 요구사항

### 1. 관리자 대시보드 (메인 화면)

#### 경로
- `/admin` 또는 `/admin/dashboard` - 관리자 대시보드

#### 목적
- 매장 운영 현황 한눈에 파악
- 주요 지표 및 주문 현황 확인

#### 레이아웃 구조 (와이어프레임 기반)

```
┌─────────────────────────────────────────┐
│ Header                                  │
│ [COZY]              [주문하기] [관리자] │ (관리자 활성)
│ ─────────────────────────────────────  │ (초록색 테두리)
├─────────────────────────────────────────┤
│ 관리자 대시보드                          │
│ ┌─────────────────────────────────────┐ │
│ │ 총 주문 1 / 주문 접수 1 /            │ │
│ │ 제조 중 0 / 제조 완료 0              │ │
│ └─────────────────────────────────────┘ │
├─────────────────────────────────────────┤
│ 재고 현황                               │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐│
│ │아메리카노│ │아메리카노│ │ 카페라떼 ││
│ │  (ICE)   │ │  (HOT)   │ │          ││
│ │          │ │          │ │          ││
│ │  10개    │ │  10개    │ │  10개    ││
│ │          │ │          │ │          ││
│ │  [+] [-] │ │  [+] [-] │ │  [+] [-] ││
│ └──────────┘ └──────────┘ └──────────┘│
├─────────────────────────────────────────┤
│ 주문 현황                               │
│ ┌─────────────────────────────────────┐ │
│ │ 7월 31일 13:00                       │ │
│ │ 아메리카노(ICE) x 1                  │ │
│ │ 4,000원              [주문 접수]    │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

#### 섹션 1: 관리자 대시보드 (주문 통계)

**요구사항**

1. **통계 표시 형식**
   - 한 줄에 모든 통계 표시
   - 구분자: " / " (슬래시)
   - 형식: "총 주문 N / 주문 접수 N / 제조 중 N / 제조 완료 N"

2. **통계 항목**
   - 총 주문: 전체 주문 건수
   - 주문 접수: "접수" 상태 주문 수
   - 제조 중: "제조 중" 상태 주문 수
   - 제조 완료: "제조 완료" 상태 주문 수

3. **실시간 업데이트**
   - 주문 상태 변경 시 자동 업데이트
   - 폴링 또는 WebSocket (선택사항)

4. **디자인**
   - 둥근 모서리 박스
   - 연한 회색 배경 또는 흰색 배경
   - 회색 테두리
   - 패딩: 16px

**상태 관리**
- 주문 통계 데이터 (React Query)
- 실시간 업데이트 (폴링 또는 WebSocket)

#### 섹션 2: 재고 현황

**요구사항**

1. **재고 카드 구조**
   - 각 메뉴별 재고 카드
   - 3열 그리드 레이아웃 (데스크톱)
   - 모바일: 1열 또는 2열

2. **재고 카드 컴포넌트**

각 카드에 포함될 정보:
- 메뉴명 (예: "아메리카노 (ICE)")
- 현재 재고 수량 (예: "10개")
- 증감 버튼 (+ / -)

3. **재고 조정 기능**
   - "+" 버튼: 재고 증가
   - "-" 버튼: 재고 감소
   - 즉시 서버에 반영
   - 최소값: 0 (음수 불가)

4. **디자인**
   - 둥근 모서리 박스
   - 각 카드 독립적으로 표시
   - 버튼: 작은 크기, 연한 회색 배경

**기능 요구사항**

1. **재고 조정**
   - 클릭 시 즉시 서버 업데이트
   - 성공/실패 피드백
   - 낙관적 업데이트 (Optimistic Update)

2. **재고 표시**
   - 현재 수량 강조 표시
   - 부족 경고 (예: 5개 이하 시 빨간색)

3. **실시간 동기화**
   - 다른 관리자가 수정 시 자동 반영 (선택사항)

**상태 관리**
- 재고 데이터 (React Query)
- 로컬 상태 (임시 수정 값)

#### 섹션 3: 주문 현황

**요구사항**

1. **주문 목록 표시**
   - 최신 주문순 정렬
   - 각 주문을 카드 형태로 표시

2. **주문 카드 구조**

각 주문 카드에 포함될 정보:
- 주문 일시 (예: "7월 31일 13:00")
- 주문 항목 (예: "아메리카노(ICE) x 1")
- 주문 금액 (예: "4,000원")
- 액션 버튼 (예: "주문 접수")

3. **주문 상태별 표시**
   - 접수 대기: "주문 접수" 버튼
   - 제조 중: "제조 중" 표시 또는 "제조 완료" 버튼
   - 제조 완료: "픽업 완료" 버튼 또는 완료 표시

4. **디자인**
   - 둥근 모서리 박스
   - 각 주문 카드 구분
   - 버튼: 오른쪽 정렬

**기능 요구사항**

1. **주문 접수**
   - "주문 접수" 버튼 클릭 시
   - 주문 상태를 "접수"로 변경
   - 고객에게 알림 전송 (선택사항)

2. **주문 상태 변경**
   - 접수 → 제조 중 → 제조 완료 → 픽업 완료
   - 각 단계별 버튼 제공

3. **주문 필터링**
   - 상태별 필터 (접수 대기, 제조 중 등)
   - 시간별 필터 (오늘, 이번 주 등)

4. **실시간 업데이트**
   - 새 주문 자동 표시
   - 주문 상태 변경 시 즉시 반영

**상태 관리**
- 주문 목록 데이터 (React Query)
- 선택된 필터
- 실시간 업데이트

## 공통 컴포넌트

### 1. Header (헤더) - 관리자 화면

**레이아웃** (와이어프레임 기반):
```
┌─────────────────────────────────────────┐
│ [COZY]              [주문하기] [관리자] │ (관리자 활성)
│ ─────────────────────────────────────  │ (초록색 테두리)
└─────────────────────────────────────────┘
```

**요소**:

1. **로고 영역 (왼쪽)**
   - "COZY" 텍스트 (또는 OrderBean 로고)
   - 어두운 회색 박스 배경
   - 하단에 초록색 테두리

2. **액션 버튼 (오른쪽)**
   - "주문하기" 버튼
     - 클릭 시 사용자 화면으로 전환
   - "관리자" 텍스트/버튼
     - **활성 상태 표시** (어두운 테두리 또는 배경색)
     - 현재 관리자 화면임을 시각적으로 표시

**디자인 스펙**:
- 관리자 버튼 활성화: 어두운 테두리 또는 배경색 변경
- 나머지는 사용자 화면과 동일

- 버튼: 작은 크기, 연한 회색 배경

### 4. 주문 카드 컴포넌트

**구조**:
```
┌─────────────────────────────────┐
│ 7월 31일 13:00                  │
│ 아메리카노(ICE) x 1              │
│ 4,000원          [주문 접수]    │
└─────────────────────────────────┘
```

**스타일**:
- 배경: 흰색
- 테두리: 회색, 1px
- 둥근 모서리: 8px
- 패딩: 16px
- 버튼: 오른쪽 정렬

## UI/UX 가이드라인

### 디자인 원칙

1. **명확성 (Clarity)**
   - 통계 숫자 명확히 표시
   - 액션 버튼 명확한 라벨
   - 상태 정보 시각적으로 구분

2. **효율성 (Efficiency)**
   - 자주 사용하는 기능 쉽게 접근
   - 최소 클릭으로 작업 완료
   - 키보드 단축키 지원 (선택사항)

3. **피드백 (Feedback)**
   - 모든 액션에 즉각적인 피드백
   - 상태 변경 시 시각적 표시
   - 성공/실패 메시지

### 색상 팔레트 (와이어프레임 기반)

- **Primary/Accent**: 초록색 (#28A745) - 로고 하단 테두리
- **Background**: 흰색 (#FFFFFF) - 메인 배경
- **Card Background**: 흰색 또는 연한 회색 (#F5F5F5)
- **Text Primary**: 어두운 회색 (#333333)
- **Border**: 회색 (#DEE2E6)
- **Button Background**: 연한 회색 (#F5F5F5)
- **Active State**: 어두운 테두리 또는 배경색 - 관리자 버튼 활성화

### 상태 표시

- **접수 대기**: 노란색 또는 주황색
- **제조 중**: 파란색
- **제조 완료**: 초록색
- **재고 부족**: 빨간색 (5개 이하)
